# iOS 悬浮窗辅助线使用说明

本文档用于说明本 iOS App 的实际使用流程、颜色含义、PiP（画中画）显示含义及常见注意事项。

## 1. 功能概览

- 实时识别
- 在 PiP 小窗中显示击打偏差数值与路线辅助。

## 2. 启动流程（必看）

1. 进入 `选择方案` 页面。  
2. 先选择：
   - `方案类型`（如 TIAN / WORLD / SOUL / DOU / HZ）
   - `使用模式`（横屏/竖屏，仅在该方案支持时显示）
   - `训练方式`（八球等）
3. 点击 `开启悬浮窗`。
4. 首次会弹出录屏/广播相关权限提示，按系统提示允许。
5. 如系统弹出广播选择器，选择对应广播扩展（你当前环境是“广播到台喵”）。
6. PiP 出现后即可开始实时分析。

## 3. 颜色与状态说明

## 3.1 主画面（识别叠加）

- 绿色矩形：当前识别到的桌面区域。
- 绿色圈（带红心）：空心白圈识别结果。
- 红色圈（带红心）：实心球识别结果。
- 白色主线：当前击打方向线（优先白线取点，失败回退球心连线）。
- 浅蓝/青色折线：翻袋路线（反射路线）。
- 角袋 1/4 圆弧（首帧调试图中可见）：用于角袋交点判定。
- 实际袋口（首帧调试图）：橙色外点 + 白色内芯（内缩偏移后的真实判定袋口）。

## 3.2 PiP 左侧数值颜色（最重要）

- `绿色`：满足击打条件（可打）。
- `红色`：不满足击打条件（不可打）。
- `--`：关键目标未识别到（如空心白圈/目标球缺失）或当前帧无法给出有效判定。

> 你当前项目规则已统一为：满足条件显示绿色，不满足显示红色。

## 3.3 PiP 左侧文字

- `直线`：第一个交点即满足条件。
- `翻袋`：经反射路径判定后满足条件。

## 3.4 PiP 右侧小矩形（路线示意）

- 白色线：当前主击打方向段（起始段）。
- 青色线：反射前主段（交点前）。
- 紫色线：反射后段（交点后）。
- 黄色点：当前选中的最终袋口点。
- 黄绿色小点：已识别实心球中心点（缩略示意）。


## 4. 使用注意事项

- 方案、横竖屏、训练方式必须与实际游戏画面一致；否则桌面矩形和识别会偏。
- 录屏/广播过程中切后台或系统中断，可能导致分析停止或延迟。
- 关闭 PiP 后，建议按提示在控制中心同时关闭：
  - 屏幕录制
  - 广播推流
  否则下次可能出现“开启失败”。
- 若发现明显漂移，优先检查：
  1. 方案是否选错  
  2. 横竖屏是否选错  
  3. 游戏画面是否有强动画遮挡（礼物、全屏特效）

## 5. 常见问题排查

- 现象：点击开启悬浮窗无反应/失败  
  处理：先完全停止录屏和广播，再重新点击 `开启悬浮窗`。

- 现象：PiP 有画面但数值一直 `--`  
  处理：通常是关键目标缺失（空心白圈或目标球），检查当前杆位与遮挡。

- 现象：PiP 线条明显延迟  
  处理：先排除系统负载/直播推流压力，再看方案匹配是否正确。

- 现象：首帧图与实时体感不一致  
  处理：首帧图用于调试当帧结果，优先看当前方案参数与识别链路是否匹配。
